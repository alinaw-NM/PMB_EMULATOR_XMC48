## Project Overview

Embedded firmware for the **Infineon XMC4700-F144x2048** (Cortex-M4) demonstrating the MULTICAN_CONFIG DAVE app with CAN gateway and FIFO functionality. The project is managed in **DAVE IDE 4.5** (Eclipse CDT with ARM-GCC).

## Build System

This project uses Eclipse CDT's Managed Build — there is no standalone Makefile. Builds must be triggered through:
- **DAVE IDE**: Project → Build Project
- **Eclipse headless build** (command line): `eclipse -nosplash -application org.eclipse.cdt.managedbuilder.core.headlessbuild -import <path> -build <project>/<config>`

Three build configurations exist: `Debug`, `Release` (size-optimized, `-Os`), and `MISRA`.

**Toolchain:** ARM-GCC with FPv4-SP-D16 (soft-fp ABI)  
**Linker script:** `linker_script.ld` — targets FLASH at `0x08000000` (2MB), PSRAM_1 (96KB) + DSRAM_1/2 (128KB each), stack 2048 bytes

**Key compiler defines:** `XMC4700_F144x2048`

**Include paths:**
- `Dave/Generated` — DAVE-generated headers
- `Libraries/XMCLib/inc` — XMC peripheral driver API
- `Libraries/CMSIS/Include` — ARM CMSIS core
- `Libraries/CMSIS/Infineon/XMC4700_series/Include` — device headers

## Architecture

### Layers

```
main.c
  └── DAVE_Init()           ← Dave/Generated/DAVE.c — initializes all DAVE apps in order
        ├── CLOCK_XMC4_0   ← System PLL to 144MHz
        ├── CPU_CTRL_XMC4_0
        ├── GLOBAL_CAN_0   ← CAN module clock gating
        ├── MULTICAN_CONFIG_0 ← Message object configuration
        ├── DIGITAL_IO_0   ← LED GPIO (output high = success)
        └── INTERRUPT_0    ← CAN Rx ISR routing
```

### DAVE Generated Code (`Dave/Generated/`)

**Do not manually edit files in this directory.** They are regenerated by DAVE IDE from the `.dconfig` model. Changes belong in `Dave/Model/` via the DAVE GUI.

Each DAVE app has three file pairs:
- `<app>.c/h` — driver wrappers
- `<app>_conf.c/h` — instance configuration structs (generated from GUI settings)
- `<app>_extern.h` — extern declarations for cross-app access

The global handle `MULTICAN_CONFIG_0` provides access to all 5 message objects (MOs) via `lmobj_ptr[]`.

### CAN Message Object Layout

| Index | Role |
|-------|------|
| MO0 | Gateway source object |
| MO1 | Gateway destination base object |
| MO2 | (supporting) |
| MO3 | **Tx object** — application writes here to transmit |
| MO4 | **Rx FIFO object** — ISR reads here on receive |

Gateway path: MO3 transmits → received by MO0 (gateway source) → forwarded to MO1 (gateway destination base) → received by MO4 via FIFO → `CAN_IRQHandler()` fires.

### Application Flow (`main.c`)

1. `DAVE_Init()` — initializes all peripherals
2. Two CAN messages are loaded into MO3 and transmitted sequentially
3. Main loop spins until `CAN_ISR_Count == 2` (both messages received via ISR)
4. FIFO CUR pointer validation (via `MOFGPR` register)
5. Tx/Rx data comparison byte-by-byte
6. LED turned on via `DIGITAL_IO_0` on success; hangs in `while(1)` on any error

Error handling is intentional `while(1)` spin loops — suitable for debug breakpoints.

## Key Libraries

- **XMCLib** (`Libraries/XMCLib/`) — Infineon's peripheral abstraction for XMC4700; CAN driver is `xmc_can.c`. Use `XMC_CAN_*` APIs.
- **CMSIS** (`Libraries/CMSIS/`) — ARM standard headers; device header is `XMC4700.h`.
- **Newlib** (`Libraries/Newlib/syscalls.c`) — minimal C stdlib syscall stubs (no OS).
